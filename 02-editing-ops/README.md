# 第02章 · 编辑操作与文本对象 —— 真正的 Vim 效率

> **一句话钩子**：第01章你学会的是「怎么在 Vim 里活下去」。这章你要学的是「为什么那么多人离不开 Vim」。
> 当你发现 `ci"` 能在一秒内把 `"old value"` 改成 `"new value"`，`:%s/foo/bar/g` 能批量替换整个文件，
> 你就再也回不去普通编辑器了。

这章的核心是**文本对象（text objects）**——Vim 最被低估、却最能改变你编辑体验的概念。

---

## TL;DR

> **30 秒速读**：Vim 的编辑命令是「动词+名词」语法——`ci"` 改引号内容、`dap` 删段落、`:%s/foo/bar/g` 全局替换。
> 
> **如果只记一件事**：学会 `ci"`（改引号内文本），这是 Vim 编辑效率的起点。

## 1. 删除家族：`x dw dd D`

Vim 的删除有十几种，初学者先掌握这 5 个：

| 键 | 删除什么 | 光标停哪 |
|----|----------|----------|
| `x` | 删**一个字符**（光标所在） | 下一字符移到光标位置 |
| `dw` | 从光标删到**下一个词开头** | 删除后光标停在剩余文本开头 |
| `dd` | 删**整行** | 下一行上移 |
| `D` | 从光标删到**行尾** | 光标停原行（现在是空行/移到下行） |
| `d$` | 和 `D` 完全一样（`d` + `$` 移动） | 同上 |

```
文本:  hello world foo
       ^ 光标在 h

x   →  ello world foo       （删了 h）
dw  →  world foo            （删了 hello + 空格）
D   →                       （从 h 删到行尾，整行没了）
```

> 💡 **规律**：`d` 是「删除动词」，后面跟任何「移动键」就是「删到那里」。
> `dw` = 删到下一个词，`de` = 删到词末尾，`d$` = 删到行尾，`dG` = 删到文件尾。这就是 Vim 的「动词+名词」语法。

## 2. 复制粘贴：`yy yw p P`

Vim 把「复制」叫做 **yank**（用 `y`），因为 `c`（copy）已经被 change 占用了。

| 键 | 作用 | 记忆 |
|----|------|------|
| `yy` | 复制**整行** | **y**ank **y**ank（双击 y 表示行操作） |
| `yw` | 复制**一个词**到下一个词开头 | yank word |
| `y$` | 复制到**行尾** | yank to end |
| `p` | 粘贴到光标**后**（行复制则粘到下方） | **p**aste |
| `P` | 粘贴到光标**前**（行复制则粘到上方） | 大写 = 反方向 |

```
文本:  hello world
       ^ 光标在 h

yy p →  hello world      （整行复制后粘到下方）
        hello world

yw p →  hellohello world （复制 "hello "，粘到光标后）
```

### Vim 的寄存器（简化版）

Vim 不只有一个「剪贴板」。删除和复制的内容会进不同的「寄存器」：

- **无名寄存器 `"`**：最近一次删除/复制的默认去向。
- **数字寄存器 `0-9`**：`0` 是最近复制，`1-9` 是最近删除的历史。
- **系统剪贴板 `+` / `*`**：和系统 GUI 共享（复制到浏览器、其他应用）。

复制到系统剪贴板：`"+y`（选中后 `"+y` 复制到 `+` 寄存器）。
从系统剪贴板粘贴：`"+p`。第03章会教你配置让 `y/p` 直接用系统剪贴板。

> ⚠️ **常见坑**：在 Vim 里 `dd` 删了一行，想 `p` 粘回来——但如果你又删了别的，无名寄存器被覆盖了。
> 用 `:reg` 查看所有寄存器，用 `"1p` `"2p` 找回更早的删除。

## 3. 替换：`r R cw`

| 键 | 作用 | 是否进 Insert |
|----|------|---------------|
| `r{字符}` | 替换**一个字符**为指定字符 | 否，瞬间完成 |
| `R` | 进入**替换模式**，逐字符覆盖 | 是（按 Esc 退出） |
| `cw` | 从光标**改到词末尾**（删词+进 Insert） | 是 |

```
文本:  hello world
       ^ 光标在 h

rx   →  xello world      （把 h 换成 x）
R    →  XY...llo world   （进入替换模式，输入 XY 覆盖 he）
cw   →  | world          （删了 hello，进 Insert，输入新词）
```

> 💡 `cw` 是 `dw` + `i` 的组合：删掉一个词，同时进入插入模式，让你直接打新词。
> 这是 Vim 编辑效率的核心——**删除和重写一步完成**。

## 4. 撤销重做：`u Ctrl-r U`

| 键 | 作用 |
|----|------|
| `u` | **撤销**上一次改动（可连续按多次） |
| `Ctrl-r` | **重做**（撤销的撤销） |
| `U` | 撤销**整行**的所有改动（恢复该行到最初状态） |

```
操作:  i输入A → Esc → i输入B → Esc → i输入C → Esc
       u   → 撤销 C，剩 AB
       u   → 撤销 B，剩 A
       Ctrl-r → 重做 B，剩 AB
       Ctrl-r → 重做 C，剩 ABC
```

> ⚠️ Vim 的撤销是**按操作粒度**的：每次离开 Insert 模式算一个「改动块」。
> 所以频繁按 Esc 不只是习惯，还能让你的撤销更精细。

## 5. 搜索：`/ n N ? :noh`

### 5.1 基本搜索

| 键 | 作用 |
|----|------|
| `/pattern` | 向**下**搜索 pattern，回车确认 |
| `?pattern` | 向**上**搜索 pattern |
| `n` | 跳到**下一个**匹配（同方向） |
| `N` | 跳到**上一个**匹配（反方向） |
| `*` | 向下搜索**光标下的词**（不用打字！） |
| `#` | 向上搜索光标下的词 |
| `:noh` | 清除搜索高亮（但保留搜索状态） |

```
操作:  /function
       n  n  n  n     连续跳到下一个 function
       N               往回跳一个
       :noh            关掉黄色高亮（但还在 function 搜索状态）
```

> 💡 `*` 是被低估的利器：光标放在变量名上，按 `*` 立刻定位所有同名变量。
> 比 `/变量名<CR>` 快得多。

### 5.2 搜索选项（本章 init.lua 配置）

| 选项 | 作用 |
|------|------|
| `incsearch` | 输入 `/fo` 时实时跳到第一个匹配（不用等回车） |
| `hlsearch`  | 所有匹配黄色高亮 |
| `ignorecase` | 忽略大小写（`Foo` 能匹配 `foo`） |
| `smartcase`  | 配合 ignorecase：搜索词有大写时恢复区分大小写 |

`ignorecase` + `smartcase` 是黄金组合：

- `/foo` → 匹配 `foo` `Foo` `FOO`（全小写，忽略大小写）
- `/Foo` → 只匹配 `Foo`（有大写，严格匹配）

## 6. 替换：`:s/old/new/`

替换命令的语法是 `:[范围]s/old/new/[标志]`。

| 命令 | 作用 |
|------|------|
| `:s/old/new/` | 替换**当前行第一个** old 为 new |
| `:s/old/new/g` | 替换**当前行所有** old 为 new（**g**lobal） |
| `:%s/old/new/g` | 替换**整个文件所有** old（`%` = 所有行） |
| `:%s/old/new/gc` | 整个文件替换，**每个都问确认**（c = confirm） |

```
文件内容:
  foo bar foo
  foo baz foo

:%s/foo/qux/g    →  qux bar qux
                    qux baz qux

:%s/foo/qux/gc   →  会逐个问: 'foo' 替换为 'qux' (y/n/a/q/l)
                    y=是 n=否 a=全替换 q=退出 l=替换后退出
```

> 💡 分隔符不一定要用 `/`。如果 pattern 里有 `/`（如文件路径），可以换分隔符：
> `:%s#/usr/local#/opt#g` 等价于 `:%s/\/usr\/local/\/opt/g`，但可读性强得多。

## 7. 文本对象：Vim 的「动词+名词」语法

这是本章的高潮。**文本对象**让你用两三个键精确选中一段文本。

### 7.1 语法结构

Vim 的编辑命令遵循「**动词 + 名词**」结构：

```
[动词] + [数量] + [文本对象/移动]
  ciw    =  c  +  iw   =  change inside word
  dap     =  d  +  ap   =  delete around paragraph
  yi"     =  y  +  i"   =  yank inside quotes
```

**动词**（5 个核心）：`c`hange、`d`elete、`y`ank、`v`isual（选中）、`>`/`<`（缩进）

**名词（文本对象）**：用 `i`（inside，内部）或 `a`（around，含包围符）开头：

### 7.2 文本对象速查表

| 文本对象 | 选中范围 | 例子（光标在 x 处） |
|----------|----------|---------------------|
| `iw` / `aw` | **词**（内部 / 含空格） | `h[x]llo` → `iw` 选中 `hello` |
| `i"` / `a"` | **双引号**内 / 含引号 | `"h[x]llo"` → `i"` 选中 `hello`，`a"` 选中 `"hello"` |
| `i'` / `a'` | 单引号内 / 含引号 | 同上 |
| `i(` / `a(` | 括号内 / 含括号 | `(x + 1)` → `i(` 选中 `x + 1` |
| `i{` / `a{` | 花括号内 / 含花括号 | `{ ... }` → `i{` 选中内部 |
| `i[` / `a[` | 方括号 | 同上 |
| `i<` / `a<` | 尖括号 | 同上 |
| `ip` / `ap` | **段落**（内部 / 含空行） | 当前段落 |
| `it` / `at` | **XML/HTML 标签**内 / 含标签 | `<div>x</div>` → `it` 选中 `x` |
| `iw` 组合 | - | `daw` 删一个词含空格，`diw` 只删词 |

### 7.3 ASCII 图解 `i"` vs `a"`

```
文本:  const name = "hello world"
                        ^^^^ 光标在 hello 里

ci" →  const name = "|"         （删了 hello world，进 Insert，光标在引号中间）

da" →  const name =             （删了 "hello world" 包括两个引号和它们之间的空格）

       ci" 只动引号内部      vs    da" 连引号一起删
```

### 7.4 实战组合

| 按键 | 效果 | 场景 |
|------|------|------|
| `ciw` | 改当前词 | 重命名变量 |
| `ci"` | 改引号内内容 | 改字符串字面量 |
| `ci(` | 改括号内内容 | 改函数参数 |
| `ci{` | 改花括号内内容 | 改函数体（小心！） |
| `dap` | 删整个段落 | 删一段文字 |
| `dat` | 删整个 HTML 标签 | 删一个 `<div>...</div>` |
| `yi(` | 复制括号内内容 | 复制函数参数列表 |
| `vi{` | 选中花括号内 | 然后可以 `>` 缩进 |

> ⚠️ **i vs a 的区别**：
> - `i"`（inside）= 只动引号**内部**的内容
> - `a"`（around）= 连**引号本身**和外围一个空格一起动
>
> 改字符串内容用 `ci"`，删掉整个字符串字面量用 `da"`。

## 8. 反模式（什么不该做）

### ❌ 删除时用 `x` 一个个删

**问题**：要删一个词却按 `x x x x x`，又慢又累。
**正确**：`dw`（删词）、`dd`（删行）、`daw`（删词含空格）。

### ❌ 改字符串时手动定位引号

**问题**：用 `dw` 或 `df"` 删字符串内容，要数有几个字符。
**正确**：`ci"`——光标在引号内任何位置都能一键选中整个字符串内容。

### ❌ 删了又删，想恢复找不回

**问题**：连按 `dd` 删了多行，`u` 只撤销最后一步。
**正确**：`u` 多按几次可以连续撤销；用 `:reg` 查寄存器，`"3p` `"4p` 找回更早删除的内容。
重要删除前可以先 `m a`（设标记 a），随时 `'a` 跳回。

### ❌ 搜索时忘了关高亮

**问题**：搜索后整个屏幕黄黄一片，分散注意力。
**正确**：`:noh`（no highlight）清除高亮，或映射快捷键 `:nnoremap <leader>h :noh<CR>`。
本章 init.lua 没设键映射（第03章才教 keymap），先用 `:noh` 命令。

### ❌ 用 `:%s/old/new/g` 裸奔替换

**问题**：替换错了不知道改了多少处，撤销只能一行一行退。
**正确**：加 `c`（confirm）标志：`:%s/old/new/gc`，每个替换前问 `y/n`。
或先 `/old` 搜索看匹配数量，心里有数再替换。

## 常见错误

> 概念懂了，实际操作还是会踩坑。这些是 Vim/Neovim 新手最常犯的错误。

| 错误 | 症状 | 解决 |
|------|------|------|
| `dd` 删了行想粘回来，结果粘的是别的 | 粘贴了错误的内容 | 用 `"0p` 粘回最近 yank 的内容；用 `:reg` 查看所有寄存器 |
| 搜索后整个屏幕黄灿灿关不掉 | 搜索高亮一直显示，分散注意力 | 输入 `:noh` 回车清除高亮（不改变搜索状态） |
| `:%s/foo/bar/g` 替换了一堆不该换的 | 全局替换失控，想撤销很麻烦 | 加 `c` 标志变成 `:%s/foo/bar/gc`，每个替换都问你确认 |
| `ciw` 改完词后不知道怎么退出插入模式 | 一直在插入模式里打字，想移动光标 | 改完后按 `Esc` 回正常模式，再用 `w/b/e` 移动 |
| 复制了内容粘贴时位置不对 | 粘到了错误的地方 | `p` 粘到光标后/下，`P` 粘到光标前/上；先确认光标位置再粘 |
