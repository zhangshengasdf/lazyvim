# 第01章 · 模态编辑 —— Vim 的第一道门槛

> **一句话钩子**：第一次打开 Vim，你按下 `i` 没反应，按下 `:wq` 屏幕上全是字母，最后只能暴力关掉终端——
> 这不是 Vim 难用，而是你还没理解**模态编辑（modal editing）**。跨过这道门槛，
> 你会发现键盘上的每个键都被重新设计过，十指不离开主键位就能完成所有编辑。

这章是整个教程的地基。后续 19 章的快捷键、文本对象、插件配置，全部建立在「模式」这个概念之上。

---

## TL;DR

> **30 秒速读**：Vim 有 4 种模式（正常/插入/可视/命令），默认是正常模式，按 `i` 进入插入模式，按 `Esc` 回到正常模式。
> 
> **如果只记一件事**：不确定的时候按 `Esc`，它永远把你送回正常模式。

## 本章目标

学完本章，你将能够：

1. **区分四种模式**：Normal / Insert / Visual / Command，知道当前在哪个模式、怎么切换。
2. **用基本键移动**：`hjkl` 四方向、`w/b/e` 按词跳、`gg/G` 跳首尾、`0/$` 行首尾、`{/}` 按段落跳。
3. **六种方式进入插入模式**：`i/a/o/I/A/O`，知道什么场景该用哪个。
4. **保存与退出**：`:w` `:q` `:wq` `:q!` `ZZ` `ZQ` 的区别，再也不会「卡在 Vim 里出不来」。
5. **打开与读取文件**：`:e` 编辑新文件，`:r` 把别的文件内容插入当前文件。

## 1. 为什么要有「模式」？

普通编辑器（VS Code、记事本）是**无模式**的：你敲什么键，屏幕上就出现什么字符。
Vim 反其道而行——默认情况下，你敲的键**不会**变成文字，而是**命令**。

```
普通编辑器：  按键 → 直接输入字符
Vim：        按键 → 先判断当前模式 → 模式决定按键的含义
```

好处是什么？**主键位的字母键被解放出来当命令用**。比如在 Normal 模式按 `d` 是删除、按 `x` 是删一个字符、
按 `w` 是跳一个词——完全不用去够方向键、鼠标、Ctrl 组合键。熟练后编辑速度远超普通编辑器。

坏处是**学习曲线陡**：第一次用完全不知道怎么打字。这章就是帮你过这道坎。

## 2. 四种核心模式

Vim 有四种你每天都会用到的模式：

| 模式 | 作用 | 怎么进入 | 怎么退出 | 左下角提示 |
|------|------|----------|----------|-----------|
| **Normal** | 执行命令（移动、删除、复制） | 默认模式；按 `Esc` 回到这里 | （本身就是默认） | 无提示 |
| **Insert** | 像普通编辑器一样输入文字 | `i` / `a` / `o` 等 | `Esc` 回 Normal | `-- INSERT --` |
| **Visual** | 选中文本（类似鼠标拖选） | `v`（字符）/ `V`（行）/ `Ctrl-v`（块） | `Esc` 回 Normal | `-- VISUAL --` |
| **Command** | 输入冒号开头的命令（保存、退出、替换） | `:`（Normal 模式下） | `Esc` 或回车执行完 | 命令行在底部 |

> ⚠️ **黄金法则**：**不确定的时候按 `Esc`**。`Esc` 永远把你送回 Normal 模式，Normal 模式是安全屋。
> 新手卡住时，先按两下 `Esc`，再决定下一步。

### 模式切换图

```
              ┌─────────────────────────────────┐
              │                                 │
              ▼                                 │
        ┌──────────┐   i/a/o   ┌──────────┐     │
        │  Normal  │ ────────► │  Insert  │     │
        │  (默认)  │ ◄──────── │          │     │
        └──────────┘    Esc    └──────────┘     │
       │  │  │                                 │
       v  V  :                                Esc
       │  │  │                                 │
       ▼  ▼  ▼                                 │
    ┌──────┐ ┌──────────┐                      │
    │Visual│ │ Command  │ ──── 执行完 ─────────┘
    └──────┘ └──────────┘
       │          │
       └────┬─────┘
            ▼
          回 Normal
```

## 3. 基本移动：不碰方向键

### 3.1 四方向：`h j k l`

为什么不用方向键？因为方向键在键盘右下角，手指要离开主键位。
Vim 把 `h j k l` 映射成左下上右——手指不用动。

```
         k  (上)
         ↑
    h ←  +  → l
     (左) ↓    (右)
         j  (下)
```

记忆口诀：
- `h` 在最左边 → 左
- `l` 在最右边 → 右
- `j` 像个向下的钩子（"jump down"）→ 下
- `k` 在 j 上面 → 上

### 3.2 按词跳：`w b e`

一个字符一个字符移动太慢，`w/b/e` 让你按「词」跳：

| 键 | 作用 | 记忆 |
|----|------|------|
| `w` | 跳到**下**一个词开头 | **w**ord |
| `b` | 跳到**上**一个词开头 | **b**ack |
| `e` | 跳到下一个词**末尾** | **e**nd |

```
文本:  The quick brown fox
       ^  ^     ^     ^
       ├─ w ─┤     │
       当前 ─w─► quick ─w─► brown
              ◄─b─
```

> 💡 大写 `W B E` 跳「空白分隔的大词」，小写 `w b e` 跳「标点分隔的小词」。
> `hello, world` 用 `w` 会停在 `,`，用 `W` 直接跳到 `world`。

### 3.3 跳到文件首尾和行首尾

| 键 | 作用 |
|----|------|
| `gg` | 跳到文件**第一行** |
| `G` | 跳到文件**最后一行** |
| `0` | 跳到**当前行最左**（含缩进前） |
| `^` | 跳到当前行**第一个非空字符** |
| `$` | 跳到**当前行最右** |

```
        gg
        ↓
   ┌───────────────────┐
 1 │   第一行           │
 2 │   ...              │
   │   ...              │
 N │   最后一行         │ ◄── G
   └───────────────────┘

行内：  0 ──► ^first_char ...... last_char$ ◄── $
        ↑                              ↑
       行首(第0列)                    行尾
```

### 3.4 按段落跳：`{ }`

代码和文章里，空行分隔「段落」。`{` 跳到上一段开头，`}` 跳到下一段开头。
在函数之间快速穿梭非常有用。

```
function a() {      ◄── { 跳到这里
  ...
}                   ◄── } 从这里跳到下一个空行后

                    ◄── 空行（段落分隔）

function b() {      ◄── } 跳到这里
  ...
}
```

## 4. 进入插入模式的六种方式

`i/a/o` 加上对应大写，一共六种进入插入模式的方式。区别在于**光标停在哪儿**：

| 键 | 光标位置 | 典型场景 |
|----|----------|----------|
| `i` | **当前字符前**插入 | 在单词中间补字 |
| `I` | **行首**（第一个非空字符）插入 | 给这行加缩进/注释 |
| `a` | **当前字符后**追加 | 在行尾继续写 |
| `A` | **行末**追加（最常用） | 在这行最后加内容 |
| `o` | **下方开新行**并插入 | 快速起新一行 |
| `O` | **上方开新行**并插入 | 在当前行前面插一行 |

```
当前行:    hello wor|d           （| 是光标）

i  →  hello wor|d     光标停在 d 前，往左插
I  →  |hello word     光标跳行首非空
a  →  hello word|     光标移到 d 后
A  →  hello word|     光标跳行末（无论原来在哪）
o  →  hello word      下方开新行，光标在新行
     |                 （原来的下一行被推下去）
O  →  |               上方开新行，光标在新行
     hello word
```

> 💡 **最常用的三个**：`i`（中间补字）、`A`（行末追加）、`o`（另起一行）。
> 记住这三个，覆盖 80% 场景。

## 5. 保存与退出：再也不怕「卡在 Vim 里」

经典笑话：程序员被 Vim 卡住只能重启电脑。学会下面这些，你就不会再被困住。

### 5.1 基本保存退出命令

在 **Normal 模式**下按 `:` 进入命令行，输入以下命令回车执行：

| 命令 | 作用 | 记忆 |
|------|------|------|
| `:w` | **保存**（write），不退出 | **w**rite |
| `:q` | **退出**（quit），仅当没改动时 | **q**uit |
| `:wq` | **保存并退出** | write + quit |
| `:q!` | **强制退出，丢弃改动** | quit + **!**（强制） |
| `:w!` | **强制保存**（覆盖只读文件） | write + ! |
| `:x` | 保存并退出（与 `:wq` 类似，但有改动时才写盘） | 更高效 |

### 5.2 不按冒号的快捷写法

Normal 模式下，有两个大写双字母组合，**不需要按冒号**：

| 键 | 等价于 | 用途 |
|----|--------|------|
| `ZZ` | `:x` | 保存并退出 |
| `ZQ` | `:q!` | 不保存退出 |

> 💡 为什么推荐 `ZZ` / `ZQ`？因为不用按 Shift+冒号，手指不用移到右上角。
> 熟练后会比 `:wq` 快很多。

### 5.3 怎么判断「有没有改动」

底部状态栏如果有 `+` 号（如 `"file.txt" 10L, 200B ++1` 中的 `+`），说明有未保存改动。
这时直接 `:q` 会报错 `E37: No write since last change`，必须 `:wq`（保存）或 `:q!`（丢弃）。

## 6. 打开与读取文件

### 6.1 `:e` —— 编辑（打开）文件

在 Neovim 内部打开另一个文件：

```vim
:e path/to/file.txt     " 打开相对路径文件
:e %:h/tab.js           " %:h 表示当前文件所在目录
:e!                     " 放弃当前文件改动，重新从磁盘加载
:e                      " 不带参数，刷新当前文件
```

### 6.2 `:r` —— 读取（插入）文件内容

把另一个文件的**内容**读进来，插入到当前光标**下方**：

```vim
:r header.html          " 把 header.html 的内容插到光标下方
:r !date                " 把 shell 命令 date 的输出插进来
:r !ls -la              " 把目录列表插进来（很有用！）
```

> ⚠️ 区分 `:e` 和 `:r`：
> - `:e` **切换**到新文件（当前文件被替换）
> - `:r` **合并**新文件内容到当前文件

## 7. 反模式（什么不该做）

### ❌ 用方向键移动

**问题**：每次移动都要把手移到方向键再移回来，效率低。
**正确**：用 `hjkl` 和 `w/b/e`，养成手不离主键位的习惯。
**例外**：玩游戏或极少数场景可以破例，写代码时请克制。

### ❌ 用鼠标点位置

**问题**：Vim 的精髓是键盘流，用鼠标会打断节奏。
**正确**：用 `/搜索词`（第02章）或 `数字 + 移动键`（如 `10j` 下移10行）快速定位。

### ❌ 一直按住方向键 / 一直按 `l`

**问题**：按 `l` 走到行尾，按 `j` 下到底部——动作太碎。
**正确**：用 `$`（行尾）、`G`（文末）、`{` `}`（段落）一次跳到位。
记住 Vim 的哲学：**用最少的按键到达目的地**。

### ❌ 插入模式下用方向键移动光标

**问题**：在 Insert 模式里长时间编辑，用方向键调整位置——容易忘记自己在 Insert 模式，
然后按 `:w` 会把 `:w` 当文字输进去。
**正确**：要移动就按 `Esc` 回 Normal 模式，移动完再 `i/a/o` 进 Insert。
理想编辑流是：**Normal 移动 → 短暂 Insert → Esc 回 Normal**。

### ❌ 改完文件直接 `:q`

**问题**：以为退出会自动保存，结果改动丢失。
**正确**：退出前看状态栏有没有 `+`；养成 `:wq` 或 `ZZ` 的肌肉记忆。
重要文件可以设 `set confirm`，退出未保存文件时弹确认。

## 常见错误

> 概念懂了，实际操作还是会踩坑。这些是 Vim/Neovim 新手最常犯的错误。

| 错误 | 症状 | 解决 |
|------|------|------|
| 按了 Ctrl+S | 终端冻结，键盘输入全无反应 | 按 `Ctrl+Q` 解冻（Ctrl+S 是终端流控锁定，不是 Vim 的功能） |
| 打开 Vim 疯狂按键没反应 | 屏幕上没有文字出现 | 按 `i` 进入插入模式再打字，默认的正常模式不会输入字符 |
| 改完文件按 `:q` 退出 | 退出后发现文件没变，改动丢了 | 用 `:wq` 保存退出，或 `ZZ`（更快）；`:q!` 强制丢弃 |
| 不小心进了 Visual 模式不知道怎么退 | 选中了一堆文字，按键没反应 | 按 `Esc` 回到正常模式，Visual 模式同样用 Esc 退出 |
| 按了 CapsLock 导致输入全大写 | 在插入模式打出大写字母 | 再按一次 CapsLock 关掉，或养成用 `Esc` 回正常模式的习惯 |

## 8. 运行验证

本章配置非常基础，但仍然要确保 `lua/init.lua` 能在 headless 模式下无错加载。

### 验证 init.lua 可加载

```bash
nvim --headless -u lazyvim/01-modal-editing/lua/init.lua -c 'qa!'
echo "Exit code: $?"
# 期望: Exit code: 0，且无 Error 输出
```

### 验证选项已设置

```bash
nvim --headless -u lazyvim/01-modal-editing/lua/init.lua \
  -c "lua print('number=' .. vim.o.number)" \
  -c "lua print('tabstop=' .. vim.o.tabstop)" \
  -c 'qa!'
# 期望输出: number=true  tabstop=2
```

### 用 shared/verify.lua 检查

```bash
nvim --headless -u lazyvim/01-modal-editing/lua/init.lua \
  -c "lua verify = dofile('lazyvim/shared/verify.lua')" \
  -c "lua verify.run({ \
    {fn = verify.check_opt, args = {'number', true}}, \
    {fn = verify.check_opt, args = {'hlsearch', true}}, \
    {fn = verify.check_opt, args = {'tabstop', 2}}, \
  })" \
  -c 'qa!'
```
