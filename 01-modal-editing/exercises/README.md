## 练习 1：模式切换肌肉记忆

### 任务

打开任意文件（或新建一个空文件），依次完成以下动作，**注意屏幕底部状态栏的变化**：

1. 启动 Neovim，确认默认在 Normal 模式（底部无 `-- INSERT --` 提示）。
2. 按 `i` 进入 Insert 模式，输入 "Hello Vim"，按 `Esc` 回 Normal。
3. 按 `a` 进入 Insert 模式（光标应该在 `m` 后面），输入 "!"，按 `Esc`。
4. 按 `o` 在下方开新行，输入 "Second line"，按 `Esc`。
5. 按 `O`（大写 O）在上方开新行，输入 "First line"，按 `Esc`。
6. 按 `v` 进入 Visual 模式（底部显示 `-- VISUAL --`），用 `hjkl` 选中几个字符，按 `Esc`。
7. 按 `:` 进入 Command 模式（光标跳到底部，出现冒号），输入 `q!` 强制退出。

### 验证

做完第 1-5 步后，文件内容应该是：

```
First line
Hello Vim!
Second line
```

### 思考题

- `i` 和 `a` 的区别，用一句话描述。
- 为什么说 `Esc` 是「Vim 的安全屋」？

## 练习 2：四种移动键的对比

### 任务

准备一个至少 10 行的文件（复制一段文章或代码）。完成以下动作并观察光标位置：

1. 光标放在第一行第一列，连续按 `j` 到最后一行，再连续按 `k` 回到第一行。
2. 光标放在某个词中间，按 `w` 三次，记录光标停在哪些位置。
3. 再按 `b` 两次，光标往回跳到哪？
4. 按 `e` 一次，光标停在当前词的末尾吗？
5. 按 `0` 到行首，按 `$` 到行尾，按 `^` 到第一个非空字符。
6. 按 `gg` 到文件首行，按 `G` 到文件末行。
7. 找两个空行分隔的段落，用 `{` 和 `}` 在段落之间穿梭。

### 思考题

- `w` 和 `W`（大写）的区别是什么？什么时候该用大写？
- `0` `^` `$` 三个行内移动键，分别在什么场景用？

## 练习 3：保存退出的组合拳

### 任务

新建一个文件 `practice.txt`，练习各种保存退出：

```bash
nvim -u lazyvim/01-modal-editing/lua/init.lua practice.txt
```

按顺序完成：

1. 按 `i` 输入一些文字，按 `Esc`。
2. 输入 `:w` 回车（只保存，不退出）。用 `ls -la` 在另一个终端看文件大小是否变化。
3. 再按 `i` 修改一些内容，按 `Esc`。输入 `:q` 回车——**会报错**，因为没保存。
4. 输入 `:q!` 回车——强制退出（改动丢弃）。重新打开文件，确认刚才的改动没了。
5. 再输入文字，这次用 `ZZ`（大写 Z 两次，不按冒号）保存退出。
6. 再打开，输入文字，用 `:wq` 保存退出。对比和 `ZZ` 的区别。

### 思考题

- `:wq` 和 `:x` 有什么细微区别？（提示：考虑文件没改动的情况）
- `:q!` 和 `ZQ` 哪个更快？为什么？

## 练习 4：`i I a A o O` 六种插入方式的差异

### 任务

准备这样一行文本（光标在 `w` 上）：

```
   hello world
```

按下表依次测试每种插入键，**记录光标停在哪个位置**，然后按 `Esc` 撤销输入，再测下一个。
（提示：按 `u` 可以撤销上一次改动）

| 键 | 测试方法 | 光标应该停在哪？ |
|----|----------|------------------|
| `i` | 光标移到 `w`，按 `i` | `w` 前面 |
| `I` | 光标移到 `w`，按 `I` | 行首（第一个非空字符 `h` 前） |
| `a` | 光标移到 `w`，按 `a` | `w` 后面 |
| `A` | 光标移到 `w`，按 `A` | 行末（`d` 后面） |
| `o` | 光标移到 `w`，按 `o` | 下方新行行首 |
| `O` | 光标移到 `w`，按 `O` | 上方新行行首 |

### 思考题

- 如果你在写代码，要在某行行末追加一个分号，最快的方式是哪个键？
- 想在当前行**前面**插一行注释，用哪个键最快？

## 练习 5：`:e` 和 `:r` 的实战

### 任务

1. 创建两个文件：`a.txt`（内容 "这是文件 A"）和 `b.txt`（内容 "这是文件 B"）。
2. 用 Neovim 打开 `a.txt`：

   ```bash
   nvim -u lazyvim/01-modal-editing/lua/init.lua a.txt
   ```

3. 输入 `:e b.txt` 回车——**当前缓冲区切换到 b.txt**。
4. 再输入 `:e a.txt` 切回 a.txt。
5. 输入 `:r b.txt` 回车——**b.txt 的内容被插入到 a.txt 光标下方**。此时 a.txt 应该是：

   ```
   这是文件 A
   这是文件 B
   ```

6. 试试 `:r !date`——把当前日期插入光标下方。

### 思考题

- `:e` 和 `:r` 的本质区别？（提示：一个是切换缓冲区，一个是合并内容）
- 怎么用 `:r` 快速把一段 shell 命令的输出（如 `ls`）插入到笔记里？

## 参考答案要点

### 练习 1 思考题

- `i` 在当前字符**前**插入，`a` 在当前字符**后**追加。
- 因为 `Esc` 永远把你送回 Normal 模式，而 Normal 是所有操作的起点。新手卡住时先按 Esc，思路就清晰了。

### 练习 2 思考题

- `w` 按标点分隔的小词跳（`hello,` 算两个词：`hello` 和 `,`），`W` 按空格分隔的大词跳（`hello,` 算一个词）。
  写代码时通常用小写 `w`（精确到每个符号），写文章时用大写 `W`（按单词跳）。
- `0` 跳第 0 列（含前导空格），`^` 跳第一个非空字符（代码里有缩进时用），`$` 跳行尾（加内容用 `A`）。

### 练习 3 思考题

- `:x` 只在文件有改动时才写盘（更新 mtime 更精确），`:wq` 总是写盘（即使没改动也会更新 mtime）。
  日常用 `:x` 或 `ZZ` 更优雅。
- `ZQ` 更快——不用按 Shift+`;`（冒号），两个 Z 直接按。但 `:q!` 更直观，新手先用 `:q!`。

### 练习 4 思考题

- 行末加分号：`A;`（`A` 进 Insert 到行末，输入分号，Esc）。
- 行前插注释：`O# `（`O` 上方开新行，输入 `# `）。

### 练习 5 思考题

- `:e file` 把当前窗口**切换**到新文件（原文件留在缓冲区，可 `:b N` 切回）；`:r file` 把文件内容**合并**到当前缓冲区。
- `:r !ls` 把目录列表插入；`:r !cat /etc/hostname` 插主机名；`:r !git log --oneline -5` 插最近 5 条提交。

## 如何使用本章代码

本章的重点是**动手练 Vim 命令**，lua/ 目录只包含基础选项（行号、高亮搜索、缩进），没有插件。
配置本身很简单，但建议先加载它再做练习，体验和裸 Neovim 的区别：

```bash
# 用本章配置启动 Neovim，然后在里面做上面的练习
nvim -u lazyvim/01-modal-editing/lua/init.lua somefile.txt

# 验证 init.lua 能正常加载（headless 模式）
cd lazyvim/01-modal-editing
nvim --headless -u lua/init.lua -c 'qa!'
# 预期：退出码 0，无报错
```

做完所有练习后，进入 [第02章 编辑操作与文本对象](../02-editing-ops/)。
