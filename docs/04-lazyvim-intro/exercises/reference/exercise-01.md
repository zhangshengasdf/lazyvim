# 练习 1 参考答案 — 对比 Neovim 配置方案

| 需求场景 | 选择 | 理由 |
|----------|------|------|
| (a) 完全自己选插件、从零开始 | **原生 Neovim** | 你想要完全掌控，不想被预设的插件集和默认值束缚。自己写 `init.lua`，自己选 lazy.nvim 或 packer.nvim，自己挑每一个插件。代价是前期投入大。 |
| (b) 从 VS Code 转过来、开箱即用 | **LazyVim** | LazyVim 的默认值已经是社区最佳实践，装完就有 LSP、补全、文件树、模糊搜索。你只需要学 Vim 操作，不用先折腾配置。 |
| (c) 想理解每一行配置，但不要太多插件 | **Kickstart.nvim** | Kickstart 是单文件 `init.lua`，每个插件都有注释解释为什么装。它只是一个起点（~10 个插件），之后你自己决定加什么。 |

**核心区别回顾**：

- **原生 Neovim** = 空地（你自己盖房子）
- **Kickstart.nvim** = 样板房（小而精，教你结构）
- **LazyVim** = 精装房（拎包入住，可局部改造）
- **AstroNvim/NvChad** = 也是精装房（风格不同）
