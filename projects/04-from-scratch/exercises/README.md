# 项目 4 练习 — 从零手搓迷你 Neovim 配置

本项目有 3 道思考题。它们不是"写代码"的练习，而是"想清楚"的练习。
花时间认真回答，你会对 Neovim 配置有更深的理解。

## 思考题 1：LazyVim 为你省了什么？

### 背景

你刚从零搭建了一套迷你配置：init.lua + 5 个 spec 文件，大约 165 行 Lua。
这套配置能用，但功能远不如 LazyVim 完整。

### 任务

打开 LazyVim 的源码仓库（`~/.local/share/nvim/lazy/LazyVim/lua/lazyvim/plugins/`），
数一数它有多少个 spec 文件、多少行 Lua 代码。

然后回答：

1. **代码量对比**：LazyVim 的插件配置有多少行？你的配置有多少行？
   比例是多少？（比如 LazyVim 是你的 10 倍？20 倍？）

2. **功能清单**：列出 LazyVim 提供了但你的配置没有的功能。至少列 10 个。
   提示：想想 which-key、状态栏、文件树、格式化、Snippet、Git blame……

3. **快捷键清单**：LazyVim 注册了多少个快捷键？你的配置注册了多少个？
   提示：在 LazyVim 源码中搜索 `vim.keymap.set` 和 `keys = {`

4. **你省了什么**：如果你要从零配置到 LazyVim 的功能水平，大概需要多少时间？
   一周？一个月？

### 预期收获

你会意识到 LazyVim 不是"偷懒"，而是"工程效率"。
一个人从零配置到 LazyVim 的水平，需要理解 50+ 个插件的 API、
处理各种边界情况、持续跟进插件更新。LazyVim 的维护者帮你做了这些。

## 思考题 2：从零配置的优缺点

### 背景

你刚体验了从零搭建配置的过程。现在退后一步，客观评价这个选择。

### 任务

填写下面的表格，每个格子至少写 3 点：

| 维度 | 从零配置的优点 | 从零配置的缺点 |
|------|--------------|--------------|
| **学习** | ？ | ？ |
| **控制** | ？ | ？ |
| **维护** | ？ | ？ |
| **启动速度** | ？ | ？ |
| **社区支持** | ？ | ？ |

### 进阶思考

1. **"理解每一行代码"真的重要吗？**
   你开车时理解发动机原理吗？你用手机时理解操作系统吗？
   什么情况下"理解底层"是必要的，什么情况下是过度工程？

2. **"没有黑盒"真的可能吗？**
   你的配置依赖 lazy.nvim、telescope.nvim、nvim-lspconfig……
   这些插件对你来说不也是黑盒吗？从零配置只是把黑盒从 LazyVim 换成了插件本身。

3. **"更轻量"真的更快吗？**
   LazyVim 用了 50+ 个插件，你用了 10 个。
   但如果 LazyVim 的懒加载做得更好，它的启动时间可能反而更短。
   启动时间取决于插件数量还是懒加载策略？

### 预期收获

没有"绝对好"的选择，只有"适合场景"的选择。
从零配置适合学习和特殊需求，LazyVim 适合日常生产力。

## 思考题 3：何时选哪个？

### 背景

你现在有了两套配置方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| LazyVim | 开箱即用、功能完整、社区活跃 | 黑盒、升级可能破坏自定义 |
| 从零配置 | 完全控制、理解每一行 | 维护负担重、功能有限 |

### 任务

针对以下 6 个场景，选择"用 LazyVim"还是"从零配置"，并说明理由：

| 场景 | 你的选择 | 理由 |
|------|---------|------|
| 1. 你是 Neovim 新手，想快速上手 | ？ | ？ |
| 2. 你只用 Neovim 写 Markdown 和笔记 | ？ | ？ |
| 3. 你在大公司工作，需要统一的团队配置 | ？ | ？ |
| 4. 你是 Neovim 插件开发者 | ？ | ？ |
| 5. 你有 RSI（重复性劳损），需要极致的按键优化 | ？ | ？ |
| 6. 你的电脑只有 512MB 内存 | ？ | ？ |

### 进阶思考

1. **混合方案**：能不能用 LazyVim 作为基础，然后在上面深度定制？
   这种方案的优缺点是什么？

2. **渐进式迁移**：能不能先用 LazyVim，然后逐步替换掉你不需要的部分？
   最终会不会变成"用 LazyVim 的框架，但从零配置每个插件"？

3. **Neovim 发行版生态**：除了 LazyVim，还有 NvChad、AstroNvim、LunarVim 等。
   它们和"从零配置"相比，各有什么定位？

### 预期收获

你会形成自己的"配置哲学"——不是别人说哪个好就用哪个，
而是根据自己的需求、能力和场景做出有理有据的选择。
